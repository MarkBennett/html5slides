A Brief History

  • The Jetpack project started in Mozilla Labs in 2009
  • goal is to simplify add-on development:
    • no XUL
    • high-level JavaScript APIs
    • packaging tool
    • integrated test framework
    • extensibility via modules using CommonJS format.

SDK 1.1!

  • 1.0 Released June, 2011
  • 1.1 released Sept 13, 2011:
    • stabilization release
    • compatibility fixes for Firefox 7+

Current State

  • Goal is to allow easy implementation of simple add-ons
  • Missing functionality can be implemented via modules
  • 1.0 contains a useful but incomplete range of modules
  • Obvious advantages:
    • Always restart-less
    • Test framework
    • Future-proof!

Roadmap

  • Top priorities: Mobile, Electrolysis, Localization
  • Secondary priorities:
    • better support for existing add-on developers
    • SDK simplification
    • missing pieces: Prefs, Sidebar, Add-on Tab, Tab Groups
    • more missing pieces: minimal / packed xpis, restart-less testing
    • Hug the web harder.

Published roadmap: https://wiki.mozilla.org/Jetpack/Roadmap

Addon-kit Module overview

  • Web content
    • windows, tabs, pag-mod
  • User interface & interaction
    • widget, panel, selection, context-menu
  • OS integration
    • notifications, selection, clipboard, hotkeys
  • Services & data:
    • request, simple-storage, page-worker
  • Utilities:
    • timers, passwords, private-browsing

Let's Get set up!

Local development

Requirements:

  • install Python ( for Windows users )
  • a text editor with good javascript support

Set-up

  • download the SDK and unzip, or
  • check out the source from Github
  • in a console window:
    • cd to /path/to/SDK
    • source bin/activate

Online developer with Builder:

  • Requirements:
    • Firefox
    • Builder 'helper' add-on
    • AMO account
  • Features:
    • in-browser development
    • create and share re-usable modules
    • planned: publish to AMO
